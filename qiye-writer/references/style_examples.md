## 1. 开头范例

**问题切入型：**
> 昨天在 WSL2 里跑 Claude Code，卡得我怀疑人生。
> 一个回车下去，等半天才反应过来。
> 折腾了一下午，最后找到了解决方案，记一下。

**折腾故事型：**
> 故事是这样的。
> 我想在 Windows 上搞一个完整的 AI 开发环境。
> Java 是我的老本行，但 Claude Code 这玩意只能在 Linux 或 macOS 上跑。
> 于是，WSL2 就成了我的唯一出路。

**发现切入型：**
> 前两天刷 GitHub 的时候，发现了一个有意思的项目。
> obra/Superpowers，115K 星。
> 看了一眼 README，我直接坐直了。

**踩坑切入型：**
> 配了一下午 Claude Code 的 Skill，最后发现原因居然是一个文件路径写错了。
> Windows 的路径分隔符，真的是程序员的噩梦。
> 记一下这个坑，希望后来人别再踩了。

**对比切入型：**
> 之前一直用 nvm 管 Node 版本，用着也还行。
> 直到有人给我推荐了 fnm。
> 用了两天，我直接把 nvm 卸了。

特点总结：永远从你正在做的一件事切入。第一句话就让读者知道你要聊什么，产生「然后呢」的冲动。

## 2. 口语化转场

**好的转场：**
- 「配置完了，来看看实际效果。」
- 「说到这个，我想起一个相关的坑。」
- 「回到 Claude Code 这块，它的 Skill 系统其实比想象中强大。」
- 「这块需要注意一下，WSL2 的路径跟 Windows 原生是不一样的。」
- 「顺着上面的思路，再来聊聊 Agent 的构建。」
- 「原理讲完了，来点实际的。」
- 「你以为配完就能用了？没那么简单。」
- 「写着写着，突然觉得这个东西跟 xxx 挺像的。」
- 「突然想起了一个之前踩过的坑。」

**要避免的转场：**
- 「接下来，让我们讨论...」
- 「值得注意的是...」
- 「在深入探讨之前，我们需要先了解...」
- 「下面来介绍一下...」

## 3. 技术知识「随手掏」示例

**好的方式（讲 WSL2 性能问题时顺带掏出文件系统原理）：**
> WSL2 的文件系统有个特点，它跑的是一个轻量级 Hyper-V 虚拟机。
> 所以跨文件系统的 IO 操作，性能会差很多。
> 也就是说，你的项目代码放在 Windows 的 C 盘，但用 WSL2 里的命令去访问，这中间隔了一层文件系统的转换。

知识不是「下面介绍 WSL2 的文件系统架构」，而是「聊着聊着自然提到了」。

**好的方式（讲 Docker 时顺带掏出容器化思想）：**
> 容器这个东西，其实核心思想特别简单。
> 你把它想象成一个标准化的打包盒。
> 不管你是 Java 应用、Python 脚本还是 Node 服务，装进盒子里，到谁的机器上都能直接跑。
> 不用关心对方装了什么环境、什么版本。

**坏的方式（千万不要这样）：**
> 下面我们来简单介绍一下 Docker 的基本概念。Docker 是一个开源的应用容器引擎，基于 Go 语言，遵从 Apache2.0 协议...

## 4. 自嘲和自我暴露

**自嘲：**
- 「折腾了一下午，最后发现原因就一行配置。我人傻了。」
- 「说实话我也不确定这是不是最佳实践，但至少它能跑。」
- 「作为一个 Java 老兵来搞 AI，很多概念我刚开始也是一脸懵。」
- 「这个方案可能不是最优雅的，但对我来说够用了。」
- 「可能有更好的做法，但我目前还没摸索到。」

**真实脆弱：**
> 从 Java 后端转 AI 全栈，说完全不焦虑是假的。
> 每天看着新工具、新框架、新模型往外蹦，有时候真的觉得学不过来。
> 但转念一想，焦虑也没用，不如打开终端写两行代码。

> 说实话写这篇文章的时候，我自己也还在摸索 Agent 的最佳实践。
> 以下的方法论是我目前跑通的一套流程，不一定是终态，但至少是一个起点。

承认不足和展示真实的学习过程，比假装全知更能建立信任。

## 5. 踩坑与排查

**好的踩坑叙事：**
> 配好之后一跑，直接报错。
> `Error: ENOENT: no such file or directory, open '/mnt/c/Users/...'`
> 一开始我以为路径写错了，改了半天发现是 Windows 路径分隔符的问题。
> WSL2 里要用正斜杠 `/`，但 Windows 默认是反斜杠 `\`。
> 就这个玩意，折腾了我半个多小时。

**层层排查型：**
> 第一次跑，成功了。
> 第二次跑，也成功了。
> 第三次跑，挂了。
> 我当时就愣住了，一模一样的命令，怎么第三次就不行了？
> 看日志，发现是 token 过期了。
> 这玩意居然有有效期，官方文档里也没写啊。

核心：踩坑过程要真实，有具体的报错信息，有你的真实反应，有排查的思路。读者看的就是这个。

## 6. 亲自下场 / 折腾记录

**配置 Claude Code 环境：**
> 说干就干。
> 先装 WSL2，选 Ubuntu 22.04。
> 装完之后第一件事，`wsl --set-default-version 2`。
> 然后进 Ubuntu，先把基本工具装上，git、curl、build-essential。
> 接着装 Node.js，这里我用的 fnm，比 nvm 快太多了。

**测试多个 AI 编程工具：**
> Spec-Kit 先试的。
> GitHub 官方出品，想着应该不会太差。
> 跑了一下，结构很清晰，但灵活度差点意思。
> 然后又试了 OpenSpec，比 Spec-Kit 轻量一些，但文档太少了，很多用法得自己猜。
> 最后是 Superpowers，好家伙，115K 星不是白来的。

核心模式：你不是在想象这些东西，你真的在自己电脑上跑了。读者跟着你一起经历。

## 7. 技术升维

**WSL2 配置 → 工具链哲学：**
> 其实折腾开发环境这件事，本质上是在做一件事，就是降低你写代码时的摩擦力。
> 每一次你要切换工具、手动配置、处理兼容性问题，都是在增加摩擦力。
> 而一个好的开发环境，应该是让你忘记环境的存在，直接进入心流状态。

**Docker → 标准化思维：**
> 容器化的思想，往大了说，其实是软件工程对「确定性」的追求。
> 在你的机器上能跑，在我的机器上也得能跑。
> 这个看似简单的需求，在容器技术出现之前，困扰了整个行业很多年。

**AI 编程 → 开发者角色变化：**
> 用了半年的 AI 编程工具，我越来越觉得，开发者的角色在发生一个根本性的变化。
> 以前你是写代码的人，现在你更像是指挥 AI 写代码的人。
> 你需要想的不是「这行代码怎么写」，而是「这个需求怎么拆解才能让 AI 准确执行」。
> 这其实是更高层次的能力。

模式：从具体的技术问题，自然连接到更大的技术视角。不是硬凑，是「聊着聊着想到了」。

## 9. 句式断裂

用极短的句子独立成段，制造停顿和重点感：

> 能跑了。

> 就一行配置。

> 折腾了一下午。

> 坑就在这里。

> 报错了。

> 就这？？？

> 跑通了。

不能每段都这样，但在关键节点（跑通的那一刻、发现坑的那一刻）用一下，效果极强。

## 11. AI 初稿 vs Qiye 风格修改对比

**AI 写的**：WSL2 是 Windows Subsystem for Linux 的缩写，它允许用户在 Windows 上运行 Linux 环境。WSL2 采用了 Hyper-V 虚拟化技术，提供了完整的 Linux 内核。

**Qiye 改的**：WSL2 这玩意，说白了就是在 Windows 里塞了一个真正的 Linux。不是那种虚拟机，是内核级别的。跑起来之后你根本感觉不到自己在 Windows 上。

差异：口语化更彻底（「这玩意」「塞了一个」），有个人感受（「根本感觉不到」），删除了教科书式的定义。

**AI 写的**：在配置过程中，可能会遇到一些常见的问题。例如路径配置错误、权限不足等。建议仔细检查配置文件，确保路径正确。

**Qiye 改的**：配到这里的时候，大概率会报错。别慌。
最常见的坑就是路径问题。Windows 的反斜杠和 Linux 的正斜杠，这俩玩意永远在打架。
我当时的解决方案是直接全改成正斜杠，一劳永逸。

差异：有具体场景（「大概率会报错。别慌。」），有真实感受（「永远在打架」），有明确的解决方案（「全改成正斜杠」）。

**AI 写的**：Claude Code 是 Anthropic 推出的一款 AI 编程助手，它支持多种开发环境，可以帮助开发者提高编码效率。

**Qiye 改的**：Claude Code，我现在的主力编程工具。
用了半年多了，从最开始的好奇，到现在的离不开。
它的 Skill 系统是真的好用，你可以把常用的工作流定义成 Skill，然后一句话调用。
怎么说呢，就像给 AI 装了一个肌肉记忆。

差异：有真实使用时长（「半年多」），有个人态度（「主力」「离不开」），有类比（「肌肉记忆」）。

**AI 写的**：接下来我们将介绍 uv 这个 Python 包管理工具的安装和使用方法。uv 是一个用 Rust 编写的超快 Python 包管理器。

**Qiye 改的**：说到 Python 的包管理，必须聊一下 uv。
这玩意是 Rust 写的，速度快得离谱。
以前用 pip 装个依赖要等半天，换 uv 之后基本是秒装。
我现在的做法是，能 uv 的场景绝不用 pip。

差异：口语化转场（「必须聊一下」），有对比感受（「秒装」），有明确态度（「能 uv 绝不用 pip」）。
